# Personal Telegram AI Agent — стартовый набор

Разверни личного ИИ-помощника на Claude Code, который живёт на твоём сервере и
общается с тобой через Telegram. Пишешь или шлёшь голосовые — он отвечает,
ищет, пишет тексты и код, помнит тебя и твои проекты.

## Что это и зачем

Обычный ИИ в чате (Claude, ChatGPT в браузере) — как консультант по телефону:
советует, но сам ничего не делает и мало что помнит. **Этот агент — тот же ИИ,
но с руками и памятью**, живущий на твоём сервере:

- **делает, а не только советует** — создаёт файлы, пишет и запускает код, строит сайты/боты;
- **помнит** — тебя, твои проекты, предпочтения;
- **самостоятельный** — разбивает задачу на шаги и выполняет;
- **всегда под рукой** — в привычном Telegram, текстом или голосом.

Это твоя личная система: память и файлы — на твоём сервере, мозг (Claude) — под
твоей подпиской. Не общий сервис.

## Что нужно заранее

1. **Сервер** — VPS на Ubuntu 22/24 (любой хостинг), доступ по SSH под root.
2. **Подписка Claude** — аккаунт с Claude Max/Pro (у оператора свой, не чужой!).
3. **Telegram-бот** — создать у [@BotFather](https://t.me/BotFather) (`/newbot`), получить токен.
4. **Свой Telegram ID** — узнать у [@userinfobot](https://t.me/userinfobot).
5. (опционально) **Groq API key** — бесплатно, для голосовых. См. `docs/SETUP-KEYS.md`.

## Установка (5 минут)

На своём компьютере:
```
git clone https://github.com/<твой-аккаунт>/agent-starter.git
cd agent-starter
scp -r . root@<IP_СЕРВЕРА>:/root/agent-starter
```
На сервере (по SSH, под root):
```
cd /root/agent-starter
sudo bash install.sh
```
Установщик спросит имя оператора, Telegram ID, токен бота (и Groq-ключ, если есть),
**создаст отдельного пользователя** под агента, поставит всё нужное (Node, Claude
Code, gateway, зависимости), развернёт агента под этим пользователем и поднимет автозапуск.

В конце — один ручной шаг (вход в Claude под пользователем агента):
```
sudo -iu <имя>          # переключиться на пользователя агента
claude                  # войти своим аккаунтом Claude, затем /exit
exit
systemctl restart <имя>-gateway
```
Готово: пиши боту в Telegram.

> Движок Telegram-шлюза — проект [qwwiwi/jarvis-telegram-gateway](https://github.com/qwwiwi/jarvis-telegram-gateway).
> Установщик клонирует его сам; этот репозиторий его не копирует.

## Что внутри

- `install.sh` — установщик (спрашивает данные, всё настраивает).
- `workspace/` — «личность» агента: CLAUDE.md / USER.md / rules.md (шаблоны с плейсхолдерами), пустая память, базовые навыки.
- `hooks/` — предохранители безопасности (блок опасных команд, защита конфигов, аудит-лог).
- `config.example.json` — конфиг шлюза (плейсхолдеры).
- `docs/SETUP-KEYS.md` — как подключить голос (Groq), веб-поиск (Perplexity), долгую память.

## Настроить под себя

После установки открой на сервере `/home/<имя>/.claude-lab/<имя>/.claude/core/USER.md`
и заполни: кто ты, чем занимаешься, как любишь работать. Агент читает это при
каждом старте — чем точнее, тем полезнее.

## Безопасность

- Агент ставится под **отдельным непривилегированным пользователем** (без пароля и
  без sudo) — он изолирован в своём `$HOME`, урон при сбое ограничен. Не под root.
- Все секреты — в `/home/<имя>/.claude-lab/<имя>/secrets/` (chmod 600), не в git.
- Хуки в `hooks/` блокируют явно разрушительные команды (`rm -rf`, `DROP TABLE`,
  force-push и т.п.) и ведут аудит-лог. Это предохранитель, не замена здравому смыслу.
